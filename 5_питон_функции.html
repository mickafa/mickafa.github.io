Функции.

Часто возникает необходимость запускать программу несколько раз с разными исходными данными. Ответ, конечно, при этом будет получаться другим. Чтобы делать это автоматически, а не писать каждый раз вручную новые исходные данные, была придумана подпрограмма - это отдельная часть программы, которая может запускаться из основной программы много раз, с разными исходными данными. В результате своей работы подпрограмма сообщает ответ. Позднее для подпрограмм был введён другой, более удачный термин - функции, которым мы и будем пользоваться. Пока мы считаем, что "подпрограмма" и "функция" - синонимы.
Функции практически представляют из себя маленькие программки-помощники, помогающие решать основную задачу.
Функции нужно уметь писать и функциями нужно уметь пользоваться.

В Питоне запись функции начинается со слова def, далее идёт имя функции, круглые скобки, знак двоеточия. В последующих строках, записываемых с отступом, содержится сама функция, и в ней один или несколько раз может встречаться return. Как только при выполнении функции встречается слово return, функция тут же завершает свою работу и сообщает основной программе в виде ответа то значение, которое записано после return. Вот и все правила. Имя функции мы можем назначить по тем же правилам, что и имя переменной, то есть любое слово. Питон понимает что это имя функции, а не переменной по наличию круглых скобок после имени. Переменная внутри скобок называется "аргумент".

Пример функции:

# функция div() возвращает наименьший делитель числа m,
# или 0, если число простое.

def div ( m ):
  d = 2
  while d * d <= m:
    if m % d == 0:
      return d
    d += 1
  return 0

 Запишите эту функцию в тетрадь "Алгоритмы", которая станет вашей коллекцией разных полезных функций. При переписывании следите за отступами.

Всё, что записано после знака #, является комментарием - это подсказка для вас. Питон не читает этот текст, но вы в тетрадь записывайте комментарии, потом будет легче вспомнить что эта функция делает.

Теперь мы можем пользоваться этой функцией. Ниже функции запишем такой пример:

n = 10
d = div ( n )
print ( n, d )

n = 17
d = div ( n )
print ( n, d )

n = 35
print ( n, div ( n ) )

 Мы получим следующий ответ:
10 2
17 0
35 5
 Первый раз ответ будет 2, потому что 2 является наименьшим делителем числа 10, функция нашла его по нашему запросу. Второй раз ответ 0, потому что число 17 простое, третий раз ответ 5. Обратите внимание, что допускается не создавать отдельную переменную для получения ответа из функции, а можно сразу же пользоваться вызовом функции, как будто это и есть вычисленное значение.

Основная теорема арифметики говорит, что любое составное число можно разложить на простые множители, причём единственным образом. Напишем программу, которая выполняет такое разложение любого натурального числа n, для чего используем готовую функцию div(). Для примера разложим число 63.

n = 63
d = div ( n )
while d > 0:
  print ( d )
  n //= d
  d = div ( n )
print ( n )

 Во второй строке мы вызовем функцию div() для получения наименьшего делителя. Если число простое, то вход в цикл не произойдёт, тогда сразу будет выполнен print() и программа завершится.
Если число n оказалось составным, то мы входим в цикл, потому что логическое условие d > 0 окажется истиным.

В цикле после print(d) разделим число n на уже известный делитель d, таким образом, дальше будем работать с числом n // d, и с ним повторим все те же операции. Таким образом, на каждой итерации цикла будем делить всё время уменьшающееся число n на очередной найденный нашей функцией делитель, пока в какой-то момент функция не вернёт 0, что будет означать, что оставшееся число уже является простым и дальше больше не делится. Произойдёт нормальное завершение цикла, потому что условие d > 0 станет ложным, и в самом конце выведем то, что осталось от числа n после всех выполненных делений.

В результате работы программы получим список простых делителей числа 63:
3
3
7
