В отличие от математики и физики, в программировании подход к решению задач совершенно другой. Здесь мы не придумываем новое решение, а первым делом смотрим какой можно применить уже готовый алгоритм. Готовые алгоритмы коллекционируем в тетради "Алгоритмы" постепенно накапливая багаж знаний. Тетрадью можно будет пользоваться на контрольной, которая предстоит через 2-3 занятия.

Не пишите сколько-нибудь сложную программу в виде сплошного кода. Всегда выделяйте маленькие вспомогательные подзадачи и поручайте эту работу функциям. Это не только уменьшает работу вам - позволяет повторно использовать уже готовый код, но и очень сильно упрощает решение задачи, потому что вам легче построить основную логику решения не вникая в мелкие детали. Готовые функции по сути уже представляют из себя готовые отдельные части решения.

1. Рассмотрим задачу: "Найти сумму простых чисел в заданном интервале." Например, в интервале от 2 до 10. Очевидно, правильным ответом будет 2+3+5+7 = 17.

У нас уже есть готовая функция div(), которая возвращает 0 если число простое, либо наименьший делитель, если число составное. Нам в задаче нужно считать простые числа, а составные нас не интересуют, их нужно просто игнорировать, поэтому нам будет удобно изменить функцию так, чтобы она возвращала 0 если число составное, либо само число, если оно простое. Чтобы не путаться изменим её имя на prime(). Тогда прогоняя цикл по всем числам интервала мы будем суммировать то, что вернёт функция prime(), и получим ответ. Обратите внимание на внесённые изменения:

def prime ( m ): # изменили имя функции на prime()
  d = 2
  while d * d <= m:
    if m % d == 0:
      return 0 # возвращаем 0, если нашёлся делитель
    d += 1
  return m # возвращаем само данное число m, если не нашлось ни одного делителя

С этой функцией решение задачи становится очень простым. Прочитайте код основной программы:

s = 0 # в начале программы создаём пустой аккумулятор s
n = 2
while n <= 10: # перебираем циклом все числа от 2 до 10
  s += prime ( n ) # добавляем в акккумулятор то, что вернёт функция prime()
  n += 1
print ( s )

 Обратите внимание, что мы на каждой итерации цикла добавляем в аккумулятор то, что вернула функция prime(), а она возвращает либо 0, если число составное, в таком случае и в аккумулятор добавляется 0, либо само число, если оно простое, в таком случае в аккумулятор добавляем это число. Таким образом к концу цикла в аккумуляторе получаем необходимую сумму.


2. Теперь рассмотрим вторую задачу: "Найти количество простых чисел в заданном интервале." Например, в этом же интервале от 2 до 10. Очевидно, правильным ответом будет 2,3,5,7 = 4.

Не изобретаем новое решение, а смотрим как нужно применить уже готовый алгоритм. На этот раз нас интересует не значение простого числа, а только сам факт его простоты. Изменим функцию prime() так, чтобы она возвращала для простых чисел 1, тогда в аккумулятор на каждой итерации цикла будет добавляться не величина простого числа, а только 1, а это как раз то, что нам и надо. Как бы один раз загибаем палец на каждое простое число.

def prime ( m ):
  d = 2
  while d * d <= m:
    if m % d == 0:
      return 0
    d += 1
  return 1 # возвращаем 1, если не нашлось ни одного делителя

# в основной части программы никаких изменений нет
s = 0
n = 2
while n <= 10:
  s += prime ( n )
  n += 1
print ( s )
