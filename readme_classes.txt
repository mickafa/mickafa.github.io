Сначала нужно взять исходный файл эксель, удалить в нём ведущие строки и столбцы, и сохранить в формате csv под именем 
"classes.csv" в формате ANSI. Также, если в какие-то дни есть 8 уроков, то эти строки тоже нужно удалить. На каждый день
должно остаться ровно по 7 строк-уроков. Если у какого класса есть 8 уроков в некий день, то это потом нужно будет
добавить вручную в итоговом html файле, перед загрузкой на сервер.

Всего должно остаться 22 столбца и 35 строк, разделитель между записями в csv файле - точка с запятой ';'

Формирование в памяти названия урока прекращается как только функция чтения встречает один из следующих символов: точка, запятая, пробел, 
открывающая скобка, точка с запятой, конец строки. А также длина имени урока не может быть длиннее, чем макроконстанта UROK_LENGTH, минус 1.
То есть 9 символов.

Название урока заканчивается по символу точки с запятой, либо по концу строки. Если название урока содержит больше символов,
чем UROK_LENGTH - 1, то остальные символы игнорируются. Поэтому в самом начале, ещё в экселе, нужно сделать массовые замены названий уроков на
адекватные, достаточно короткие и понятные имена.  

Название урока не может содержать символы: точка, запятая, пробел, открывающая скобка, точка с запятой.

Только единственный файл classes.csv нужен как входной для программы. В результате работы на экран выводится количество обработанных 
уроков (770), и создаётся комплект из файлов html, по одному для каждого класса. Они получаются в кодировке ANSI, поэтому все эти 
итоговые файлы нужно вручную по одному перекодировать в текстовом редакторе в UTF-8. 

Если какие-то классы имеют 8 уроков, то не забыть добавить 8-й урок вручную. Так получаются готовые к загрузке html файлы. 

Предполагается, что каждый день есть ровно по 7 уроков (итого, 35 уроков в неделю, возможно некоторые пустые), количество классов 22, 
порядок следования имён классов задан списком в функции write_class(). Этот порядок должен соответствовать порядку колонок в исхдной эксельке.

Сводная таблица rc.html программно не создаётся, её нужно сделать вручную, в кодировке UTF-8.

Далее полученные 22 файла, плюс оглавление rc.html можно загружать на сервер.
